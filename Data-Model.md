# CarWise 第一版数据模型

## 1. 设计目标

第一版数据模型服务 4 件事：

- 少手工录入：优先保存图片、截图、文件，再生成待确认草稿。
- 信息可追踪：每条报价、承诺、问题、费用都能看到来源和状态。
- 首页能汇总：未确认、未落实、未解决、临近提醒都能回到首页。
- 后续可扩展：第一版先服务 007GT，但数据结构不把未来车型扩展堵死。

## 2. 核心对象

第一版需要保存 10 类信息：

1. 车辆
2. 原始凭证
3. 识别任务
4. 报价
5. 销售承诺
6. 验车分组
7. 验车项目
8. 问题记录
9. 用车费用
10. 提醒
11. 时间线

其中“原始凭证”和“识别任务”是降低手工录入的关键。

## 3. 对象关系

```text
车辆
├─ 报价
├─ 销售承诺
├─ 验车分组
│  └─ 验车项目
│     └─ 问题记录
├─ 用车费用
├─ 提醒
├─ 原始凭证
│  └─ 识别任务
└─ 时间线
```

## 4. 通用字段

所有主要信息都建议包含以下字段：

| 字段 | 说明 |
| --- | --- |
| ID | 每条记录的唯一编号 |
| 创建时间 | 记录创建时间 |
| 更新时间 | 最近一次修改时间 |
| 信息来源 | 手工填写、图片识别、截图识别、文件识别 |
| 是否已确认 | 自动识别的信息需要用户确认 |
| 原始凭证 | 对应的图片、截图或文件 |
| 备注 | 用户补充说明 |

## 5. 车辆

车辆是所有记录的主对象。第一版默认只有一辆车。

| 字段 | 说明 | 第一版默认值 |
| --- | --- | --- |
| 车辆名称 | 用户给车辆起的名字 | 我的 007GT |
| 品牌 | 车辆品牌 | 极氪 |
| 车型 | 车辆型号 | 007GT |
| 年款 | 车辆年款 | 可空 |
| 外观颜色 | 外观颜色 | 紫外 |
| 内饰颜色 | 内饰颜色 | 紫内 |
| 电池版本 | 电池容量或版本 | 可空 |
| 轮毂 | 轮毂样式 | 可空 |
| 选装 | 选装配置 | 可空 |
| VIN | 车辆识别码 | 可空 |
| 生产日期 | 车辆生产日期 | 可空 |
| 提车日期 | 实际提车日期 | 可空 |
| 当前状态 | 已下定、待提车、已提车、用车中 | 待提车 |

## 6. 原始凭证

原始凭证用于保存用户上传的照片、截图和文件。它是自动识别的来源，也能作为后续维权和核对证据。

| 字段 | 说明 |
| --- | --- |
| 凭证名称 | 用户或系统生成的名称 |
| 凭证类型 | 图片、截图、PDF、文档、其他 |
| 来源页面 | 首页、购车、提车、用车、档案 |
| 文件地址 | 文件保存位置 |
| 上传时间 | 用户上传时间 |
| 凭证用途 | 报价、承诺、问题、费用、车辆文件、其他 |
| 关联对象 | 关联到报价、承诺、问题、费用、提醒等 |
| 是否已识别 | 是否已经生成识别任务 |

## 7. 识别任务

识别任务用于保存“从图片或文件里识别出了什么”。识别结果不能直接变成正式记录，必须由用户确认。

| 字段 | 说明 |
| --- | --- |
| 任务名称 | 例如“销售聊天截图识别” |
| 来源凭证 | 对应的原始图片、截图或文件 |
| 识别类型 | 报价、承诺、问题、费用、提醒、车辆信息、不确定 |
| 识别状态 | 待识别、待确认、已确认、已忽略、识别失败 |
| 候选内容 | 系统识别出的草稿内容 |
| 建议保存位置 | 报价、承诺、问题、费用、提醒、档案 |
| 用户处理结果 | 确认保存、修改后保存、拆分保存、忽略、仅保存凭证 |
| 关联正式记录 | 确认后关联到正式记录 |

## 8. 报价

报价用于记录一次门店或销售给出的购车方案。

| 字段 | 说明 |
| --- | --- |
| 报价名称 | 例如“深圳门店第一次报价” |
| 门店 | 报价门店 |
| 销售姓名 | 对接销售 |
| 报价日期 | 报价时间 |
| 官方价 | 车辆官方价格 |
| 优惠金额 | 现金优惠或价格优惠 |
| 保险费用 | 保险预估或实际金额 |
| 购置税 | 新能源车通常为 0，也保留字段 |
| 上牌费用 | 上牌相关费用 |
| 贷款方式 | 全款、贷款、未确定 |
| 补贴总额 | 已确认或预计补贴合计 |
| 落地价 | 最终预计总价 |
| 当前状态 | 草稿、已确认、已作废 |
| 原始凭证 | 报价单截图、聊天截图等 |

## 9. 销售承诺

销售承诺用于追踪赠品、补贴、权益、服务和费用减免。

| 字段 | 说明 |
| --- | --- |
| 承诺名称 | 例如“原厂量子太阳膜” |
| 承诺类型 | 赠品、补贴、权益、服务、费用减免、其他 |
| 金额或价值 | 例如 2999 元 |
| 规格说明 | 例如“4 门 + 后挡” |
| 负责人 | 销售或交付人员 |
| 预计完成时间 | 约定完成时间 |
| 实际完成时间 | 实际落实时间 |
| 当前状态 | 未确认、已确认、待落实、已落实、有争议 |
| 原始凭证 | 微信截图、确认单、交付单等 |

## 10. 验车分组

验车分组用于组织 Checklist。

| 字段 | 说明 |
| --- | --- |
| 分组名称 | 例如“文件资料核对” |
| 排序 | 页面显示顺序 |
| 完成数量 | 已完成检查项数量 |
| 总数量 | 分组内检查项总数 |
| 是否关键分组 | 是否影响签字付款前提醒 |

### 10.1 第一版默认分组

- 付款与签字前确认
- 文件资料核对
- 车辆外部检查
- 车辆内部检查
- 车机、屏幕与电子功能
- 空调、音响与用电功能
- 核心信息确认
- 随车物品确认
- 销售承诺与赠品确认
- 最后签字前

## 11. 验车项目

验车项目来自《极氪007GT 提车验车 Checklist》。

| 字段 | 说明 |
| --- | --- |
| 所属分组 | 关联验车分组 |
| 检查内容 | 具体检查事项 |
| 排序 | 分组内显示顺序 |
| 是否完成 | 是否已勾选 |
| 是否关键项 | 是否进入签字前总检查 |
| 是否发现问题 | 是否有关联问题 |
| 关联问题 | 对应的问题记录 |
| 照片 | 检查照片或问题照片 |
| 备注 | 用户补充说明 |

## 12. 问题记录

问题记录用于提车或用车中发现的问题。

| 字段 | 说明 |
| --- | --- |
| 问题标题 | 简短标题 |
| 问题描述 | 具体情况 |
| 问题照片 | 现场照片或截图 |
| 发现时间 | 发现问题的时间 |
| 发现阶段 | 提车前、提车当天、用车中 |
| 关联检查项 | 来自哪个验车项目 |
| 责任人 | 销售、交付员、售后或其他 |
| 处理方式 | 约定如何处理 |
| 预计处理时间 | 约定完成时间 |
| 实际解决时间 | 关闭问题的时间 |
| 当前状态 | 待确认、待处理、处理中、已解决、有争议 |
| 原始凭证 | 照片、聊天截图、维修单等 |

## 13. 用车费用

用车费用用于记录提车后的花费。

| 字段 | 说明 |
| --- | --- |
| 费用类型 | 充电、保险、保养、维修、贴膜、洗车美容、轮胎、上牌、其他 |
| 发生日期 | 费用发生时间 |
| 金额 | 支出金额 |
| 里程 | 当时车辆里程 |
| 商家或地点 | 费用发生地点 |
| 说明 | 费用说明 |
| 凭证 | 票据、截图、保单、维修单 |
| 是否计入总费用 | 是否参与费用统计 |
| 当前状态 | 草稿、已确认、已作废 |

## 14. 提醒

提醒用于保养、保险和其他待办事项。

| 字段 | 说明 |
| --- | --- |
| 提醒名称 | 例如“首保提醒” |
| 提醒类型 | 首保、常规保养、保险到期、年检、轮胎、其他 |
| 预计日期 | 什么时候提醒 |
| 预计里程 | 到多少公里提醒 |
| 当前状态 | 未开始、即将到期、已完成、已忽略 |
| 关联费用 | 完成后可关联实际费用 |
| 原始凭证 | 保单、保养记录、聊天截图等 |
| 备注 | 用户补充说明 |

## 15. 时间线

时间线用于把车辆经历串起来。

| 字段 | 说明 |
| --- | --- |
| 事件名称 | 例如“已下定”“完成提车验车” |
| 事件类型 | 购车、承诺、验车、问题、费用、提醒、文件、其他 |
| 事件时间 | 发生时间 |
| 说明 | 简要说明 |
| 关联对象 | 报价、承诺、问题、费用、提醒等 |
| 原始凭证 | 对应图片、截图或文件 |

## 16. 首页汇总规则

首页不单独保存一套数据，而是从已有记录里汇总。

| 首页内容 | 来源 |
| --- | --- |
| 当前车辆状态 | 车辆 |
| 下一件要办的事 | 承诺、问题、提醒、识别任务 |
| 待确认识别结果 | 识别任务 |
| 未落实承诺 | 销售承诺 |
| 提车进度 | 验车项目 |
| 未解决问题 | 问题记录 |
| 下一次提醒 | 提醒 |
| 最近时间线 | 时间线 |

## 18. 第一版数据验收标准

- 能保存一辆默认 007GT 车辆。
- 能保存原始图片、截图或文件。
- 能从原始凭证生成识别任务。
- 识别结果能保持“待确认”状态。
- 用户确认后，识别结果能变成报价、承诺、问题、费用或提醒。
- 未确认内容不会混进正式记录。
- 未落实承诺能回到首页。
- 未解决问题能回到首页。
- 临近提醒能回到首页。
- 费用、问题、承诺、提醒能进入时间线。
