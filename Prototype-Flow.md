# CarWise 第一版文字原型

## 1. 原型目标

这份原型用于确认第一版流程是否顺畅。它不追求视觉效果，只描述每个页面显示什么、主要按钮是什么、点击后去哪里。

第一版交互原则：

- 优先拍照、截图、上传文件。
- 系统先生成草稿。
- 用户确认后才保存。
- 手工填写只作为兜底。

## 2. 全局结构

### 2.1 底部导航

第一版使用 5 个主入口：

1. 首页
2. 购车
3. 提车
4. 用车
5. 档案

### 2.2 全局快捷入口

所有主页面都保留一个明显的快捷入口：

- `上传图片 / 截图`

点击后进入“识别入口”，用户选择要识别的内容类型：

- 报价 / 补贴
- 销售承诺
- 提车问题
- 费用 / 票据
- 车辆文件
- 不确定，先保存为凭证

上传后进入“识别确认页”。识别确认页不是主导航页面，是临时处理页面。

## 3. 首页

### 3.1 页面内容

首页标题：`我的 007GT`

首页显示：

- 当前状态：已下定 / 待提车 / 已提车 / 用车中
- 下一件要办的事
- 待确认识别结果
- 未落实承诺
- 提车 Checklist 进度
- 未解决问题
- 下一次提醒
- 最近时间线

### 3.2 页面按钮

- `上传图片 / 截图`
- `继续验车`
- `查看未落实承诺`
- `处理待确认结果`
- `记录费用`
- `新增提醒`
- `更新车辆状态`

### 3.3 点击路径

| 用户点击 | 进入哪里 | 结果 |
| --- | --- | --- |
| 上传图片 / 截图 | 识别入口 | 选择识别类型并上传 |
| 处理待确认结果 | 识别确认页 | 确认、修改或忽略草稿 |
| 继续验车 | 提车页 | 回到上次未完成的检查项 |
| 查看未落实承诺 | 购车页 | 只看未落实承诺 |
| 记录费用 | 用车页 | 打开新增费用入口 |
| 新增提醒 | 用车页 | 打开新增提醒入口 |
| 更新车辆状态 | 状态选择弹窗 | 切换车辆阶段 |

### 3.4 首页状态示例

```text
我的 007GT

状态：待提车
下一件要办：确认太阳膜安装时间

待确认识别结果：2 条
未落实承诺：5 条
提车进度：0 / 128
未解决问题：0 条
下一次提醒：保险生效确认，2026-05-28

最近时间线：
- 已下定
- 销售确认 8000 极氪积分
- 已上传报价截图，待确认
```

## 4. 购车页

### 4.1 页面内容

购车页分为 4 个区块：

- 报价明细
- 补贴明细
- 销售承诺
- 销售确认文字

页面优先展示已确认信息。待确认草稿单独放在顶部提醒，不混入正式记录。

### 4.2 页面按钮

- `上传报价截图`
- `上传销售聊天截图`
- `新增报价`
- `新增承诺`
- `生成确认文字`
- `复制确认文字`
- `只看未落实`

### 4.3 点击路径

| 用户点击 | 进入哪里 | 结果 |
| --- | --- | --- |
| 上传报价截图 | 识别入口 | 识别报价、补贴、落地价 |
| 上传销售聊天截图 | 识别入口 | 识别赠品、权益、服务承诺 |
| 新增报价 | 报价编辑页 | 手工补充报价 |
| 新增承诺 | 承诺详情页 | 手工补充承诺 |
| 生成确认文字 | 销售确认文字区 | 生成可复制内容 |
| 复制确认文字 | 当前页面 | 复制到剪贴板 |
| 只看未落实 | 当前页面筛选 | 只显示待落实和有争议内容 |

## 5. 销售承诺详情页

### 5.1 页面内容

承诺详情页用于处理一条具体承诺。

显示内容：

- 承诺名称
- 承诺类型：赠品 / 补贴 / 权益 / 服务 / 费用减免 / 其他
- 金额或价值
- 规格说明
- 负责人
- 预计完成时间
- 当前状态
- 截图或凭证
- 备注

### 5.2 页面按钮

- `标记已确认`
- `标记待落实`
- `标记已落实`
- `标记有争议`
- `添加截图`
- `保存`

### 5.3 点击路径

| 用户点击 | 进入哪里 | 结果 |
| --- | --- | --- |
| 标记已确认 | 当前页面 | 状态变为已确认 |
| 标记待落实 | 当前页面 | 首页待办数量增加 |
| 标记已落实 | 当前页面 | 从未落实列表移除 |
| 标记有争议 | 当前页面 | 首页显示为重点待办 |
| 添加截图 | 图片上传 | 作为证据保存 |
| 保存 | 购车页 | 返回承诺列表 |

## 6. 识别确认页

### 6.1 页面目的

识别确认页用于处理系统从图片、截图或文件中提取出来的草稿。

它的核心不是让用户重新填表，而是让用户快速判断：

- 这条识别对不对？
- 要保存到哪里？
- 有没有需要改的地方？

### 6.2 页面内容

- 原始图片或截图预览
- 识别类型
- 识别出的候选字段
- 建议保存位置
- 需要用户确认的内容

### 6.3 页面按钮

- `确认保存`
- `修改后保存`
- `拆成多条记录`
- `忽略`
- `先保存为凭证`

### 6.4 点击路径

| 用户点击 | 进入哪里 | 结果 |
| --- | --- | --- |
| 确认保存 | 来源页面 | 草稿变为正式记录 |
| 修改后保存 | 编辑页 | 改完后保存 |
| 拆成多条记录 | 拆分确认页 | 一张截图拆成多条承诺或费用 |
| 忽略 | 来源页面 | 不保存识别结果 |
| 先保存为凭证 | 档案页 | 只保存原图，不生成业务记录 |

### 6.5 识别确认示例

```text
识别确认

来源：销售聊天截图
建议保存为：销售承诺

识别结果：
- 原厂量子太阳膜，价值 2999 元，含 4 门 + 后挡
- 两次老带新，共 8000 极氪积分
- 免 499 元指标申请费

操作：
[确认保存] [修改后保存] [拆成多条记录] [忽略]
```

## 11. 关键流程

### 11.1 通过截图生成销售承诺

1. 用户进入购车页。
2. 点击 `上传销售聊天截图`。
3. 上传截图。
4. 系统识别出承诺草稿。
5. 用户进入识别确认页。
6. 用户确认或修改。
7. 承诺进入购车页。
8. 未落实承诺同步到首页。

### 11.2 提车当天发现问题

1. 用户进入提车页。
2. 按 Checklist 检查。
3. 发现问题后点击 `拍照留证`。
4. 上传或拍摄照片。
5. 系统生成问题草稿。
6. 用户确认责任人、处理方式和处理时间。
7. 问题进入首页待办。
8. 签字前总检查展示该问题。

### 11.3 通过票据生成费用

1. 用户进入用车页。
2. 点击 `上传票据 / 付款截图`。
3. 系统识别金额、日期、项目和商家。
4. 用户确认。
5. 费用进入用车页。
6. 费用同步到档案时间线。

### 11.4 处理待确认识别结果

1. 首页显示待确认数量。
2. 用户点击 `处理待确认结果`。
3. 进入识别确认页。
4. 用户逐条确认、修改、拆分或忽略。
5. 确认后的内容进入对应页面。
6. 首页待确认数量减少。

## 12. 第一版原型验收

这份文字原型通过以下标准判断是否可用：

- 用户从首页能进入 4 个核心闭环。
- 用户能通过截图生成报价或承诺草稿。
- 用户能通过拍照生成问题草稿。
- 用户能通过票据或付款截图生成费用草稿。
- 所有识别结果都有确认步骤。
- 用户能从首页看到未完成、未落实、未解决和待确认事项。
- 购车、提车、用车、档案之间的跳转路径清楚。
