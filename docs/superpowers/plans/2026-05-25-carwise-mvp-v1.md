# CarWise 007GT 第一版开发计划

> 面向后续执行者：本计划用于把《007GT 爱车管家产品方案》压缩成第一版可落地范围。第一版目标是先做可记录、可勾选、可追踪的本地网页应用，不做社区、车控、全车型库和交易功能。

**目标：** 做出一个能服务 007GT 车主从购车、提车到早期用车的第一版工具。

**第一版范围：** 只做 4 个闭环：购车报价和销售承诺管理、007GT 提车验车 Checklist、赠品/补贴/问题整改追踪、用车费用和保养提醒。

**交付形态：** 先做网页应用，可在电脑或手机浏览器里使用。第一版重点是把信息记清楚，把待办盯住，把现场提车流程跑通。

**交互原则：** 尽量少让用户手工填表。第一版优先支持拍照、截图、上传文件后自动生成候选内容，用户确认后保存。手工录入只作为兜底。

---

## 1. MVP 边界

### 1.1 第一版必须做

- 首页：展示“我的 007GT 当前状态”和最重要待办。
- 购车页：通过截图或手工方式记录报价、补贴、销售承诺、赠品和落实状态。
- 提车页：内置 007GT 提车验车 Checklist，支持勾选、拍照留证、记录问题、确认赠品。
- 用车页：通过票据、截图或手工方式记录费用、保养、保险、维修和提醒。
- 档案页：保存车辆信息、文件、照片和时间线，并支持从文件中生成待确认信息。

### 1.2 第一版明确不做

- 不做远程车控。
- 不做车友社区、帖子、评论、私信。
- 不做全车型库。
- 不做二手车交易。
- 不做复杂贷款计算器。
- 不做维修厂比价。
- 不做充电桩地图。
- 不做积分交易。
- 不做账号体系和多人协作。

### 1.3 第一版默认车辆

- 品牌：极氪
- 车型：007GT
- 外观颜色：紫外
- 内饰颜色：紫内
- 车辆状态：已下定、待提车、已提车、用车中四选一

## 2. 页面结构

### 2.1 首页：我的 007GT 当前状态

**页面目的：** 用户打开后马上知道现在最该处理什么。

**显示内容：**

- 当前车辆状态：已下定 / 待提车 / 已提车 / 用车中
- 销售承诺落实进度
- 提车清单完成进度
- 未解决问题数量
- 下一件要办的事
- 下一次提醒
- 最近时间线

**按钮：**

- `更新车辆状态`
- `新增待办`
- `继续验车`
- `查看未落实承诺`
- `记录一笔费用`
- `新增提醒`
- `上传图片或截图`
- `处理待确认识别结果`

**验收标准：**

- 用户能一眼看到车辆当前阶段。
- 用户能看到未完成承诺、未完成验车项、待整改问题和下一次提醒。
- 用户从首页能进入购车、提车、用车、档案 4 个主要页面。
- 用户能看到待确认的识别结果，并进入确认流程。

### 2.2 购车页：报价、补贴、销售承诺

**页面目的：** 把报价、补贴、赠品和销售承诺分开记清楚。

**显示内容：**

- 报价卡：官方价、优惠、保险、购置税、补贴、贷款/全款、落地价
- 补贴卡：保险补贴、置换补贴、政府新购补贴
- 销售承诺列表：承诺内容、金额/价值、规格、负责人、预计完成时间、状态
- 赠品列表：实体礼品、太阳膜、遮阳帘、门槛保护套、关门硅胶套、车主标等
- 销售确认卡：用于复制给销售确认的文字摘要
- 待确认识别结果：报价单截图、销售聊天截图、补贴说明截图生成的候选内容

**按钮：**

- `新增报价`
- `上传报价截图`
- `上传销售聊天截图`
- `编辑报价`
- `新增补贴`
- `新增承诺`
- `标记已落实`
- `标记未落实`
- `生成销售确认卡`
- `复制确认文字`

**默认数据：**

- 5000 元保险补贴
- 5000 元置换补贴
- 5000 元政府新购补贴
- 1199 元实体礼品
- 2999 元官方原厂量子太阳膜
- 两次老带新共 8000 极氪积分
- 遮阳帘、门槛保护套、关门硅胶套
- 车友会车主标
- 保养工时费 8 折
- 免费上门取送车
- 深圳上牌指标协助
- 免 499 元指标申请费

**验收标准：**

- 用户能完整录入一次购车报价。
- 用户能从报价截图生成报价草稿，并确认保存。
- 用户能把补贴、赠品、官方权益和销售承诺分开查看。
- 用户能给每条承诺设置状态：未确认、已确认、待落实、已落实、有争议。
- 用户能从销售聊天截图生成承诺草稿，并确认保存。
- 用户能生成一段可发给销售的确认文字。

### 2.3 提车页：验车清单、问题留证、赠品确认

**页面目的：** 提车当天按步骤检查，避免漏项。

**显示内容：**

- 提车当天模式入口
- Checklist 分组进度
- 当前分组检查项
- 问题记录列表
- 随车物品确认
- 赠品确认
- 签字付款前总检查
- 待确认问题草稿

**按钮：**

- `开始提车当天模式`
- `勾选完成`
- `发现问题`
- `添加照片`
- `拍照生成问题草稿`
- `新增备注`
- `标记待整改`
- `标记已解决`
- `进入下一组`
- `查看签字前检查`

**Checklist 分组：**

- 付款与签字前确认
- 文件资料核对
- 车辆外部检查
- 车辆内部检查
- 车机、屏幕与电子功能
- 空调、音响与用电功能
- 核心信息确认
- 随车物品确认
- 销售承诺与赠品确认
- 最后签字前

**验收标准：**

- 用户能按组完成 007GT 提车验车清单。
- 用户能看到总进度和每组进度。
- 用户能对任意检查项记录问题。
- 用户能先拍照留证，稍后确认成问题记录。
- 用户能把未解决问题自动转成待办。
- 用户在签字付款前能看到未完成项和未解决问题。

### 2.4 用车页：费用、保养、保险、维修记录

**页面目的：** 让新车主开始形成长期用车档案。

**显示内容：**

- 保养提醒：首保、下一次保养、保养工时费优惠
- 保险提醒：保险公司、险种、生效日期、到期日期
- 维修记录：问题、处理方式、金额、照片/文件
- OTA 记录：版本、升级时间、备注
- 待确认费用草稿

**按钮：**

- `记录费用`
- `上传票据或付款截图`
- `新增保养提醒`
- `新增保险提醒`
- `记录维修`
- `记录贴膜`
- `记录 OTA`
- `上传凭证`

**验收标准：**

- 用户能新增一条费用，并选择费用类型。
- 用户能从票据或付款截图生成费用草稿，并确认保存。
- 用户能设置首保或下次保养提醒。
- 用户能记录保险到期时间。
- 用户能查看费用列表和近期提醒。

### 2.5 档案页：车辆信息、文件、时间线

**显示内容：**

## 4. 核心流程

### 4.1 购车承诺确认流程

1. 用户上传报价单截图、销售聊天截图或补贴说明截图。
2. 系统生成报价、补贴、赠品、权益和服务承诺的候选内容。
3. 用户确认、修改或忽略候选内容。
4. 系统按类型分组展示已确认内容。
5. 用户生成销售确认卡。
6. 用户把确认文字发给销售。
7. 销售确认后，用户把状态改为已确认。
8. 提车或后续落实时，用户把状态改为已落实。

### 4.2 提车当天验车流程

1. 用户进入提车页。
2. 用户点击 `开始提车当天模式`。
3. 用户按分组逐项勾选。
4. 如果发现问题，用户先拍照留证。
5. 系统生成问题记录草稿。
6. 用户确认问题描述、责任人和处理时间。
7. 未解决问题进入首页待办。
8. 用户进入 `签字前检查`。
9. 所有关键项完成后，再签字付款。

### 4.3 赠品、补贴、问题整改追踪流程

1. 购车页建立承诺和补贴记录。
2. 提车页确认当场已交付内容。
3. 未交付内容保留为待落实。
4. 问题整改记录保留状态和负责人。
5. 首页持续显示未完成数量和下一件要办的事。

### 4.4 用车记录和提醒流程

1. 用户在用车页上传票据、保单、维修单或付款截图。
2. 系统生成费用或提醒草稿。
3. 用户确认、修改或忽略草稿。
4. 到期前首页显示提醒。
5. 完成后关联实际费用。
6. 档案页自动形成时间线。
