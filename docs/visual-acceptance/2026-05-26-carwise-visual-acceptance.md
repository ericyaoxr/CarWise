## 3. 核心流程检查

检查路径：

1. 首页。
2. 进入权益页。
3. 将第一条权益状态改为「已落实」。
4. 进入提车验车页。
5. 勾选第一个验车项。
6. 标记第二个验车项为问题。
7. 回到首页检查变化。

## 4. 流程结果

| 步骤 | 页面状态 | 验收结果 |
| --- | --- | --- |
| 首页初始 | 待办 25，验车 0%，权益 19，未解决问题 2 | 通过 |

| 勾选验车项 | 验车进度从 0% 变为 1%，完成项从 0 / 156 变为 1 / 156 | 通过 |

## 15. 问题留证与跟进闭环

本轮继续沿用本地应用，不再使用 Figma。新增目标是把“验车发现问题”推进到“记录、跟进、回看”的闭环。

已完成内容：

1. 问题记录增加“问题类型”，可选外观、内饰、屏幕、权益、交付材料、其他。
2. 问题记录支持编辑责任人、截止时间、处理状态和处理方式。
3. 从提车验车页点击“标记问题并跟进”后，会自动生成问题记录并跳转到问题留证页。
4. 首页会优先展示未解决问题，并显示当前未解决问题数量。
5. 问题页新增当前要处理的问题摘要，突出类型、责任人和截止时间。

验收路径：

首页 → 提车页 → 标记问题 → 问题留证页 → 编辑责任人和截止时间 → 保存 → 回首页查看未解决问题数。

验收结果：

- 流程通过。
- 编辑保存通过。
- 首页联动通过。
- 问题页可见“问题类型”“责任人”“截止时间”“问题记录”等关键内容。
